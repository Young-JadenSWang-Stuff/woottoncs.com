Object Oriented Programming


1) Constructors must have the same name as the class name
2) Any Exception can be thrown but IllegalArgumentException is the most common one for incorrect input
3) Constructors can have the same name but still be legal because of the different parameter rule
4) Using this.(n,50) in line 17 can be used to initialize the object without changing
5) Drives or Client Classes are only neccessary for Projects, not Homeworks
6) All HELPER functions must be set as private as the client should not be able to use these methods

File Input/Output


1) All File IO should have a try-catch catching FileNotFoundExceptions or IOExceptions
2) File Input/Output has
- .nextLine() reads every line, there is no .nextString() method
- .nextInt() reads until the next int
- .nextDouble() reads until the next double
3) To move the "cursor" where the compuer looks at each line, the scanner must be reinitialized 4) any .nextMethod drives the cursor forward
