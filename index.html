Introduction to Object Oriented Programming

- Constructors must have the same name as the class name
- Any Exception can be thrown, here are some common ones
    - FileNotFoundException
    - IOException
    - IndexOutOfBoundsException
    - IllegalArgumentException
- Constructors can have the same name but still be legal because of the different parameter rule
- Using this.(n,50) in line 17 can be used to initialize the object without changing

- Drivers or Client Classes are only neccessary for Projects, not Homeworks
- All HELPER functions must be set as private as the client should not be able to use these methods

- Class Composition is when you incorporate other classes you write into your current class

- Figure 2: Initializing Enum Values
    - An Enum File is basically a collection of constants, similar to a constant array
    - Stylistcally, all values should be capitalized as well as the Enum name itself
- Figure 3: values(), toString(), ordinal()
    - values() method stores the values inside the enum file into an array of enums
    - the enum is effecively a new data type and can be used to store an individual enum value
    - toString() method changes any value inside the enum to a string, ie. enum
    - ordinal() a sort of indexOf(), ordinal finds and returns the integer of the location of a value
File Input

- All File IO should have a try-catch catching FileNotFoundExceptions or IOExceptions
- File Input has
   - .nextLine() reads every line, there is no     .nextString() method as String is technically     an object, despite exhibiting primitive like     behaivior
   - .nextInt() reads until the next int
   - .nextDouble() reads until the next double
- To move the "cursor" where the compuer looks at each line back to the top, the scanner must be reinitialized, yes this requires another try catch

- any .next[value] Method drives the cursor forward
- if you have an infinite loop, chances are, you didnt use a .next method to drive your "cursor" forward
Resizing Arrays

- Resizing an array requires the recopying of the entire old array and placing it into a new larger one
- Create a temporary array an element larger
- Loop through your original array and copy all data into the second array
- Return the second array
Manipulating Objects

- Despite instance variables being marked private, the variables can still be accessed because the class is created and owned by you, the class programmer
- To Access the instance variable, the object must be of the same class
- Since Objects merely POINT to the location on the heap from the stack, everything that is not a primitive or a string must be deep copied. This is to avoid changing the data in array2 when editing array1

- During Class Composition, accesssors are mandatory as you may be using classes not created by you, java has no way to verify your identity
